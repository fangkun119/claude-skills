# Voice Text Correction Command

语音转录稿纠错工具，专门用于修正语音转录过程中的专业术语识别错误和中文识别错误。

## Usage

```markdown
/md-fix-voice-text "{input_file}" "{content_introduction}"
```

## Parameters

- `input_file`: 语音转录稿的文件路径
- `content_introduction`: 内容简介，描述转录稿涉及的专业领域和主要技术术语

## Output

修正后的完整内容覆盖原文档，并输出剩余识别错误的报告

## 核心功能

### 1. 错误识别与修正
- **专业术语纠错**：识别并修正特定领域的英文专业术语识别错误
- **中文错误修正**：识别并修正语音转录过程中的中文识别错误
- **上下文理解**：基于内容简介理解专业领域背景，提高纠错准确性

### 2. 内容保护
- **结构保持**：保持原文档的结构和格式
- **内容完整性**：只修正识别错误的词语，不改变实质内容
- **批量修正**：一次性修正所有识别错误

### 3. 质量控制
- **二次检查**：修正完成后进行全面检查，确保没有遗漏的错误
- **错误报告**：输出仍然存在的识别错误清单
- **覆盖更新**：直接覆盖原文档，保持文件路径不变

## 执行流程

### 第一步：错误分析
1. **领域识别**：根据内容简介识别专业领域
2. **术语收集**：收集领域内常见专业术语
3. **错误扫描**：扫描文档中可能的识别错误

### 第二步：错误修正
1. **专业术语修正**：修正英文专业术语的识别错误
2. **中文错误修正**：修正中文识别错误
3. **上下文验证**：基于上下文验证修正的准确性

### 第三步：结果输出
1. **文件覆盖**：用修正后的内容覆盖原文档
2. **二次检查**：全面检查修正结果
3. **错误报告**：输出剩余识别错误报告

### 第四步：质量保证
1. **完整性检查**：确保内容完整性和结构一致性
2. **准确性验证**：验证修正的术语和词汇准确性
3. **错误总结**：总结仍然存在的问题

## 使用示例

```markdown
/md-fix-voice-text "spring-cloud-alibaba-2024/Nacos.md" "spring cloud alibaba的,有不少关于spring cloud alibaba, nacos, java, spring, spring boot技术栈, Java分布式中间件..."
```

## 常见错误类型

### 英文专业术语错误
- 发音相近错误：`spring claud` → `spring cloud`
- 大小写错误：`nacos` → `Nacos`
- 连字符错误：`microservice` → `micro-service`

### 中文识别错误
- 同音字错误：`技术站` → `技术栈`
- 语义相近错误：`中间键` → `中间件`
- 专业术语错误：`分布式中间键` → `分布式中间件`

## 重要提醒

- **内容保护**：只修正识别错误的词语，不改变内容的实质含义
- **结构保持**：保持原文档的段落结构和格式
- **准确性优先**：不确定的错误不要修正，保持原样
- **全面检查**：修正后要进行二次检查，确保没有遗漏
- **错误报告**：如实报告仍然存在的识别错误

## 预期效果

通过此命令，可以有效修正语音转录稿中的专业术语识别错误，提高文档的可读性和专业性，特别适用于技术分享、会议记录等场景的转录稿处理。
