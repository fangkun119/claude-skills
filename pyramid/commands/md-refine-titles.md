# /note-refine-titles

Generate hierarchical titles for text content based on semantic analysis.

## Usage

```
/note-refine-titles {input_file_path}
```

## Output

优化后的完整markdown内容（UTF-8编码）直接输出到标准输出，不修改原文件。

## 核心特性

### 严格的原文保护
- **零修改原则**：绝对不修改、删除或增加任何原文内容
- **格式保持**：保持原文的所有标点符号、空格、换行等格式细节
- **语义完整**：确保原文的逻辑结构和论证链条完整无缺

### 智能语义换行
- **长段落检测**：自动识别超过3-4个句子的长段落
- **语义边界识别**：在句号、问号、感叹号等自然边界处插入换行
- **可读性优化**：通过适当的换行提高长文本的可读性，但不改变原意

### 精准的层次分析
- **主题聚类**：使用语义分析将相关内容合理分组
- **逻辑关系识别**：识别时间顺序、因果关系、重要性层次等
- **平衡性控制**：确保各级标题下的内容量相对均衡

## Requirements

1. **严格保持原文完整性** - 只添加或替换标题，**绝对不得修改、删除或增加任何原文内容**，包括文字、标点符号、格式等所有细节
2. **智能分块处理** - 对于长文档（超过2000字符），采用智能分块处理机制，确保完整覆盖所有内容
3. **语义换行优化** - 对于过长的段落（超过3-4个句子），可根据语义在适当的句子边界处插入换行符，提高可读性，但不得改变原文意义
4. 根据文档长度采用适当的层级结构，避免过度细分：
   - **短文档（<3000字符）**：仅使用##章节层级，每个章节800-1500字符
   - **中等文档（3000-10000字符）**：使用##章节和###小节，每个小节800-1500字符
   - **长文档（>10000字符）**：才考虑使用####子小节，每个子小节800-1500字符
5. 使用指定的编号模式
6. 输出结果到标准输出（UTF-8编码）
7. **重点优化**：创建合理的标题结构，确保每个标题下有足够的内容量：
   - **####子小节**：必须包含800-1500字符（相当于5-8个实质性段落）
   - **###小节**：必须包含2000-3000字符（包含3-5个相关子小节或足够的内容深度）
   - **##章节**：必须包含4000-6000字符（完整论述一个主要主题）
   - 如果某个内容块无法达到最小内容量要求，**必须**将其与语义相邻的相关内容合并，而不是强制拆分
8. **内容完整性优先原则**：优先考虑内容的逻辑完整性和实用性，而非追求标题的数量：
   - 当某个概念本身内容简洁时，将其放在更高级别的标题下
   - 避免为了达到标题数量而人为拆分连贯的内容
   - 确保每个标题都代表一个完整、有意义的主题单元

## 智能分块处理机制

### 分块触发条件
- 当输入文档超过 **15000字符** 时，自动启用智能分块处理
- 每个分块大小控制在 **15000-20000字符** 以内
- 确保有足够的处理空间，同时避免过度分割

### 分块算法
1. **语义边界检测**：优先在段落、现有章节等自然边界处分割
2. **上下文保持**：确保每个分块包含足够的上下文信息
3. **编号连续性**：维护分块间标题编号的连续性和一致性
4. **重叠处理**：在分块边界处保留少量重叠内容（50-100字符），确保语义连贯性

### 处理流程
1. **文档分析**：检测文档长度，确定是否需要分块
2. **分块分割**：按照语义边界将文档分割成多个块
3. **逐块处理**：为每个分块生成标题，维护上下文信息
4. **结果合并**：智能合并各块处理结果，重新编号标题
5. **质量验证**：确保处理结果的完整性和一致性

## Title Hierarchy Rules

- **Document title**: `# {总标题内容}` (main title for the entire document)
- **Level 1 sections**: `## 1`, `## 2`, `## 3`, etc.
- **Level 2 subsections**: `### 1.1`, `### 1.2`, `### 2.1`, etc.
- **Level 3 sub-subsections**: `#### 1.1.1`, `#### 1.1.2`, `#### 2.1.1`, etc.

## Implementation Notes

### 文本处理原则
- **严格保护原文内容**：采用只读方式处理文本，确保不修改任何文字、标点、格式
- **智能语义分析**：通过自然语言处理技术识别主题转换和逻辑边界
- **保持原文风格**：保留原作者的语言风格、表达习惯和叙述方式
- **维护内容连贯性**：确保添加标题后文本的逻辑流畅性和可读性

### 语义换行处理
- **识别长段落**：检测超过3-4个句子的连续文本块
- **寻找语义边界**：在句号、问号、感叹号等句子结束处考虑换行
- **保持逻辑完整**：确保换行不会破坏句子的完整性和连贯性
- **优化可读性**：通过适当的换行提高长文本的可读性，但不得改变原意

### 层次结构生成
- **主题聚类分析**：使用语义相似度算法将相关内容分组
- **逻辑关系识别**：识别因果关系、时间顺序、重要性层次等
- **智能平衡控制**：确保各级标题下的内容量充足且均衡，每个####子小节必须包含800-1500字符
- **内容完整性优先**：严格限制过度拆分，当语义块内容不足时，必须与相邻内容合并
- **编号一致性**：严格按照1, 1.1, 1.1.1的格式进行编号

### 质量保障措施
- **内容完整性检查**：生成后对比原文，确保无内容丢失或修改
- **格式正确性验证**：检查markdown格式和编号规则的正确性
- **语义一致性确认**：确保标题准确反映对应内容的主题
- **可读性优化**：通过合适的分段和换行提升整体可读性
- **内容量与结构质量检查**：
  - **最小内容验证**：严格检查每个####子小节是否包含至少800-1500字符，每个###小节是否包含2000-3000字符
  - **过度拆分修正**：强制修正内容不足的子小节，必须与语义相邻内容合并
  - **结构合理性评估**：评估标题层级是否与文档长度匹配，避免短文档使用过深层级
  - **实用性验证**：确保标题结构服务于内容理解，每个标题都代表完整的主题单元

### 特殊情况处理
- **长文本无标点**：对于缺少标点的长文本，根据语义主题进行合理分段
- **混合内容类型**：处理包含叙述、说明、论证等多种表达方式的文本
- **专业术语密集**：在技术文档中保持专业术语的准确性和一致性
- **口语化表达**：在口语化文本中保持自然流畅的表达方式

### 输出格式要求
- **标准输出**：将处理后的完整 markdown 内容以UTF-8编码直接输出到标准输出
- **标题层次清晰**：确保各级标题的层级关系明确
- **内容分布均匀**：避免某些部分过长而其他部分过短
- **逻辑结构完整**：保持原文的逻辑结构和论证链条
- **阅读体验优化**：通过合理的分段和换行提升阅读体验

## 具体实现步骤

### 第一步：文档分析
```markdown
请分析以下文档的长度和结构：
{{input_text}}

如果文档字符数超过15000，请启用智能分块处理模式，并说明："文档长度为X字符，将启用智能分块处理"。
如果文档不超过15000字符，请直接进行标题优化处理。
```

### 第二步：分块处理（仅在需要时执行）
```markdown
根据文档分析结果，请将文档按照以下规则分割成多个块：

1. 每个分块控制在15000-20000字符以内
2. 优先在段落边界、现有章节标题等自然边界处分割
3. 在分块边界保留50-100字符的重叠内容
4. 为每个分块添加分隔标识：=== 分块 X ===

分割后的分块格式：
=== 分块 1 ===
[分块内容]
=== 分块 1 结束 ===

=== 分块 2 ===
[分块内容]
=== 分块 2 结束 ===
```

### 第三步：逐块标题优化
```markdown
对每个分块进行标题优化处理：

处理分块 {{chunk_number}}/{{total_chunks}}：

=== 分块 {{chunk_number}} 开始 ===
{{chunk_content}}
=== 分块 {{chunk_number}} 结束 ===

要求：
1. 保持原文完整性，只添加或替换标题
2. 根据上下文调整标题编号，确保连续性
3. 继承前面分块的标题结构和编号
4. **内容量控制**：严格确保每个####子小节包含800-1500字符（5-8个实质性段落），每个###小节包含2000-3000字符
5. **强制合并**：当语义块内容少于最小要求时，必须与语义相邻的相关内容合并到同一个标题下
6. **层级简化**：根据文档长度选择合适的标题层级，避免为短文档创建过深的结构

请输出该分块的标题优化结果。
```

### 第四步：结果合并与最终优化
```markdown
将所有分块的处理结果合并，并进行最终优化：

合并规则：
1. 移除分块分隔符
2. 确保标题编号的连续性和一致性
3. 检查并修正可能的标题重复或跳跃
4. 保持整体的逻辑结构和层次关系
5. **严格内容量检查**：强制检查并修正内容不足的标题，确保所有####子小节≥800字符，###小节≥2000字符

请输出最终的完整markdown文档。
```

## 执行指令模板

当用户使用 `/note-refine-titles {input_file_path}` 时，按以下步骤执行：

1. **读取并分析文档**：检测文档长度，确定处理模式
2. **短文档处理**（≤15000字符）：直接应用标题优化规则，严格确保每个标题层级的内容量要求
3. **长文档处理**（>15000字符）：
   - 启用智能分块机制
   - 按语义边界分割文档
   - 逐块处理并维护上下文，严格控制每个标题层级的最小内容量
   - 合并结果并优化结构，强制修正内容不足的标题单元
4. **输出最终结果**：确保完整性、一致性和内容合理性

## 注意事项

1. **分块边界处理**：确保不会在句子中间分割，优先选择段落边界
2. **上下文连续性**：后一分块需要了解前一分块的标题结构和编号
3. **完整性验证**：最终结果必须包含原文的所有内容
4. **编号一致性**：标题编号必须严格遵循 1, 1.1, 1.1.1 的格式
5. **严格内容量平衡**：确保内容量充足，每个####子小节≥800字符，###小节≥2000字符
6. **实用性优先**：以内容的逻辑完整性和实用性为首要考虑，层级深度服务于内容理解
