## 1. 任务

在 note-tool 这个 plugin 里面加一个 skill,  名为 md-img-to-html。

它可以把用户传入的文档中，所有 Markdown 格式的图片，都转换为 HTML 标签格式的图片。

例如：![[图片源路径|600px]] 或者 ![替换文字|600px](图片源路径) 
会转换为：<img src="图片源路径" style="display: block; width: 600px;" alt="替换文字">

## 2. 格式转换规则（ 转换前 Markdown，转换后 HTML ）

### 2.1 图片源路径

必须严格来自 Markdown 格式的输入，绝对不可以自行更改

至于这个路径是否正确，这是输入文件编写时要考虑的，而不是这个Skill的职责，因为这个Skill只负责格式转换，它并不了解具体的应用场景，也没有足够的信息做出准确判断，所以绝对不可以自行更改、做出超越职责的举动。

### 2.2 替换文字

转换前的 Markdown 有替换文字：使用转换前的替换文字
转换前的 Markdown 无替换文字：使用默认值“替换文字”，不可以留空、必须要有替换文字

例如

转换前没有替换文字：![[图片源路径]] 或者 ![](图片源路径)
转换后使用默认的替换文字： <img src="图片源路径" style="display: block; width: 800px;" alt="替换文字">

这样做是因为，一些工作流会强制要求HTML格式的图片，必须有alt属性且不能为空，如果不使用默认值填充，会导致工作流中断

### 2.2 宽度

#### (1) 转换前的 Markdown 格式**没提供**图片宽度

Markdown格式的图片形如：![[图片源路径]] 或者 ![替换文字](图片源路径)

转换后的HTML标签格式自动添加默认宽度，分为两种情况：

① 情况1：用户没有在提示词中给出默认宽度、则使用800px

<img src="图片源路径" style="display: block; width: 800px;" alt="替换文字">

② 情况2：用户在提示词中给指定了默认宽度，则使用用户指定的默认值

例如用户在调用skill时，指定了默认宽度为100%，则使用100%为默认宽度

<img src="图片源路径" style="display: block; width: 100%;" alt="替换文字">

#### (2) 转换前的 Markdown 格式**提供了**图片宽度

Markdown格式的图片形如：![[图片源路径|600px]] 或者 ![替换文字|600px](图片源路径)

转换成HTML标签后同样要保留宽度不变

例如：

转换前： ![[图片源路径|600px]] 或者 ![替换文字|600px](图片源路径)
转换后： <img src="图片源路径" style="display: block; width: 600px;" alt="替换文字">

## 3. 转换过程

这个Skill按如下顺序转换：

1. 先梳理文件中所有的markdown格式的图片、它们的替换文字、宽度、Markdown标签在文件中的位置。
2. 列出计划，按照在文中出现的位置、从下到上逆序注意转换。逆序的原因是：先修改下面的图片，不会改变上面图片的位置，确保列出的计划依旧可行
3. 转换之后检查HTML格式中的 alt 属性，如果 alt 属性不存在或者值为空，都要用默认值补上。
