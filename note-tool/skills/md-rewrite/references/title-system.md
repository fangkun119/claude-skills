# 四级标题体系详细说明

## 概述

改写文件中所有章节标题必须严格遵循以下四级标题体系，不允许超出这四个层级。每个层级的标题编号必须按照下方规格编写。

## 设计动机

统一的标题层级保证改写文档的结构一致性。Skill 的改写流程会将文档拆分到多个 Sub Agent 并行处理，只有严格的标题规范才能确保各章节独立改写后仍能无缝拼合，不会出现层级冲突或编号错乱。

## 四级标题规范

### Level 1: 章

- **名称**: 章
- **Markdown 前缀**: `##`
- **编号模式**: 阿拉伯数字递增
- **格式**: `## {n}. `
- **示例**: `## 1. `

**说明**:
- 编号从 1 开始递增
- 每个章代表文档的主要主题板块
- 编号后跟一个点和一个空格

### Level 2: 节

- **名称**: 节
- **Markdown 前缀**: `###`
- **编号模式**: 父级.子级递增
- **格式**: `### {n}.{m} `
- **示例**: `### 1.1 `

**说明**:
- 编号由父级章编号和子级节编号组成
- 父级编号必须与当前所在的章编号一致
- 子级编号从 1 开始递增
- 格式为 `{父级}.{子级}` 后跟一个空格

### Level 3: 小节

- **名称**: 小节
- **Markdown 前缀**: `####`
- **编号模式**: 圆括号阿拉伯数字递增
- **格式**: `#### ({k}) `
- **示例**: `#### (1) `

**说明**:
- 编号用圆括号包裹
- 从 1 开始递增
- 括号后跟一个空格

### Level 4: 段落条目

- **名称**: 段落条目
- **Markdown 前缀**: `#####`
- **编号模式**: 带圈数字递增
- **格式**: `##### {circled_n} `
- **示例**: `##### ① `

**说明**:
- 使用带圈数字符号
- 支持的符号: ①-⑩, ⑪-⑳
- 符号后跟一个空格

## 附加规则

1. **不允许超出四个层级**: 任何尝试使用 `######`（Level 5+）的行为都是违规的
2. **编号必须按定义编写**: 每个层级的编号必须严格遵守上述格式规范
3. **层级关系必须正确**: 子级标题必须在父级标题之下，不能跨越层级

## 示例文档

```markdown
## 1. 概述

这是第一章的内容。

### 1.1 背景

这是第一章第一节的内容。

#### (1) 技术背景

这是小节内容。

##### ① 历史发展

这是段落条目。

##### ② 当前状态

这是另一个段落条目。

#### (2) 业务背景

这是另一个小节。

### 1.2 目标

这是第一章第二节。

## 2. 技术方案

这是第二章。

### 2.1 架构设计

这是第二章第一节。
```

## 验证工具

使用 `validate_titles.py` 脚本可以验证文档是否符合四级标题体系规范：

```bash
python scripts/validate_titles.py <markdown_file_path>
```

该脚本会检查：
- 标题编号是否正确递增
- 父子关系是否正确
- 是否有超出四级的标题
- 带圈数字是否有效
